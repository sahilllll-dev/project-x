/store/:storeName

and subdomain-style hostnames ending in:

.projectx.com

The root route / checks whether the current hostname is a store subdomain. If yes, it renders the storefront. Otherwise it redirects to /login.

New order notifications:

AppContext polls orders every 15 seconds for the selected store. If a newer order appears, it dispatches:

projectx:new-order

App.jsx listens for this event, shows a toast, and plays a short sound.

product.quantity <= product.lowStockThreshold

KNOWN RISKS / TECHNICAL DEBT

1. Backend routes are not protected by user auth middleware.
2. Backend uses Supabase service role and trusts storeId from request body/query.
3. Frontend stores current user/session in localStorage.
4. No bearer token is sent from frontend API calls to backend.
5. Product images are stored as text/base64 rather than uploaded to Supabase Storage.
6. Signup only creates Supabase user; it does not automatically create a first store.
7. Public checkout can create orders, decrement stock, upsert customers, and send notifications.
8. No tests are currently configured.
9. server/package.json has test set to "Error: no test specified".


WHAT TO DO WHEN MODIFYING THIS PROJECT

When making frontend changes:

- Check routes in src/App.jsx.
- Use src/utils/api.js for backend calls.
- Use useAppContext() for user/store/app state.
- Use useToast() for notifications.
- Preserve current SCSS styling structure unless asked to redesign.
- Store-specific features should use currentStore.id.
- Do not hardcode real Supabase keys.

When making backend changes:

- Main file is server/index.js.
- Use existing mapper functions like toProduct, toStore, toOrder, toCoupon.

PROJECT SUMMARY

Project X is a React + Express + Supabase ecommerce SaaS platform where authenticated users can create and manage multiple stores, add products, configure themes, install apps like SEO Helper, manage coupons/customers/orders, and publish a public storefront. The frontend uses React Router, context-based state, Supabase auth, and a central API wrapper. The backend is an Express server using a Supabase service role key to read/write Postgres tables for stores, products, orders, customers, coupons, themes, pages, and apps. Public customers can browse stores/products and place orders, while store owners use the protected admin dashboard to manage their business.
